# GolangStartForFreshman
首先,欢迎你加入海格星航的大家庭,这个git是由2022年3月入职的周佩瑾完成.由于本人是校招生没有工作经验同时对于go也不是很了解,在刚入职的时候踩了很多的坑.此git的存在意义是为了帮助新同事更好更快的融入我们的后端开发环境中. 如果你同样也希望为未来的同事提供帮助,你也可以提交一些代码或者培训内容. 希望我们可以在这里共同进步.

1. 首先,希望你可以了解一下go的基础语法(该链接是go的中文文档,如果你习惯于英文环境可以自行google go的英文文档.):
    https://www.topgoer.com/

2. 好的, 现在你已经是个成熟的go语言开发师了,经过几个小时的学习相信你已经信心满满了,那我们开始我们第一个项目吧!项目要求如下:

    1. 核心要求: 通过go创建一个可以用于增删改查的后端api微服务, 数据库选用mySQL,这里推荐查找用get方法,增改删均使用post方法,当然你也可以按照增加post,全部改变put,部分改变pitch, 删除delete来完成api的构造. 但是生产环境中我们是都用post实现功能.(大概是方便前端调用吧,毕竟统一写法方面前端代码复用)

    2. 测试要求: 通过postman进行测试. 在测试前请自行使用unit test进行测试. 
    unit test 指引文档: https://zhuanlan.zhihu.com/p/377834750
    同时,unit test 有一个非常好用的傻瓜工具, gotest包, gotest是一个开源的无敌好用的测试工具(可以偷懒),他可以自动生成unit test文件,推荐简单函数都使用此包进行测试增加效率.
    gotest github链接:https://github.com/rakyll/gotest

    3. demo数据库信息: 我们提供了一个数据库供你测试和使用,相关的数据库信息可以找后端的同事询问并使用,由于本文档公开所以不会在这里提供.
    
    4. 代码规范: 详情请见新员工入职指引中的代码规范的要求,希望你可以在这个demo中重视代码规范, 好看的代码会给大家精神上的愉悦不是吗?

    5. 小tips: 首先,推荐你使用beego框架搭建服务器(由于本人最开始一头雾水全靠google, 使用原生的方法完成的这个作业, 浪费了非常多的时间.), beego使我们开发环境中普遍使用的框架,该框架非常简单易懂,使用这个框架完成第一个作业之后你就可以快速的融入到我们的开发了. 其次, 推荐使用orm包来操作数据库,这也是我们在生产环境中经常使用的一个原生包. 

    6. 最后, 如果你对于代码有什么疑问, 可以参考本仓库的demo-zpj, 这个代码是由笔者完成的作业. 由于当时几乎完全靠自己google完成, 所以在各种方面都有非常大的问题. 如果你完成了一份很棒的作业, 希望你也可以将自己的代码发到这个git上来供大家参考. 最后祝你在海格星航的工作愉快!!!


3. 现在你已经熟练掌握了我们go后端的主要开发手段和开发内容. 接下来将是我们业务的重中之重: 对于交通部协议的充分了解和应用. 在协议的文件夹中有我们在开发过程中所需用到的协议. 我们的开发过程必须完全按照协议中的要求完成. 其中后端主要使用的协议为 JTT808, 请务必多次阅读并充分理解该协议.



写在最后, 首先这并不是一个非常成熟完美的文档, 该稳当的创作也是由于笔者的一时兴起.如果你对这个新人指引非常感兴趣, 希望你也可以为这个git做一些贡献.
